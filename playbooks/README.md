Playbooks are step-by-step guides for handling recurring situations. When something happens, find the relevant playbook and follow it.

## How to Use Playbooks

1. **Identify the trigger** — what situation are you in?
2. **Find the playbook** — use the index above
3. **Follow the steps** — in order, don't skip
4. **Document outcomes** — if the playbook revealed a gap, update it

---

## How to Add a Playbook

When you encounter a recurring situation that isn't documented:
1. Create a new file: `[situation-name].md`
2. Define the **trigger** (what starts this?)
3. Write **step-by-step steps** (numbered, in order)
4. Add an **escalation path** at the end
5. Add a row to the index above
6. Link from any relevant program README

## Guiding Philosophy

> Document processes early, even before they're needed at scale. The ALO nationwide launch worked because operational detail was owned from the pilot forward. The ESO pilot is the current opportunity.
