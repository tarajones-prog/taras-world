# Playbook: CITY Feature Request

**Trigger:** You need a new tag, field, label, or behavior change in CITY to support a program workflow.

---

## Step 1 — Document the Need
Before posting anything, write down:
- What triggers the need? (specific workflow or situation)
- What should CITY do today that it doesn't?
- What would the ideal behavior look like?
- How many transactions or users does this affect?

The clearer your request, the faster it gets prioritized and implemented.

## Step 2 — Post in #tech-support
Post in the `#tech-support` Slack channel with:
- Clear description of the request
- Business justification (what program, what impact)
- Ideal timeline or urgency level
- Any supporting examples

## Step 3 — Tag the Right Person
- For ESO workflows, tagging, or labeling → tag Agnes Omega
- For other CITY requests → tag the general `#tech-support` channel and wait for routing
- If urgent and no response → DM the appropriate engineer

## Step 4 — Follow Up
If no response within 2 business days, follow up in the thread or via DM.

## Step 5 — Track in Linear (if complex)
If the request requires a code change (not just a config tweak):
- File or request a Linear ticket in the **Sales & Transactions** team
- Reference your Slack thread in the ticket

## Tips

- The faster you document the need, the faster it gets built
- In-person or Slack DMs with engineers move faster than channel posts for urgent items
- "Show, don't tell" — if you can show Agnes the exact CITY screen and what you need, it helps

---

## After Implementation

- Confirm the tag/field/behavior works as expected
- Update any playbooks or process docs that reference the old behavior
- Note the win in lessons-learned if it was a meaningful process improvement
