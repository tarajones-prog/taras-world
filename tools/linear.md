# Linear — Issue and Project Tracking

**What:** Linear is Opendoor's engineering and project tracking tool. Program Managers use it to file bugs, track feature requests, and manage cross-functional projects.

---

## How PMs Use Linear

### Bug Tracking
When a CITY bug or system issue needs an engineering fix:
1. File a ticket in the **Sales & Transactions** team
2. Include: description, affected transactions (addresses/tokens), priority, impact
3. Link any supporting docs (correction sheets, Slack threads)
4. Track resolution and follow up if stalled

**Example:** TXN-316 — Cash vs. Cash Plus correction for specific property (created 2/18/26)

### Project Management
When running a cross-functional initiative:
1. Create or join a Linear project (e.g., "HSA Agent Acquisition Training" in AGENT team)
2. Break work into issues with owners and due dates
3. Use to track deliverable status and blockers
4. Reference in 1:1s and team meetings

---

## Ticket Best Practices

A good Linear ticket includes:
- **Title:** Clear, specific (e.g., "Cash Plus label incorrect for properties in batch X")
- **Description:** What's wrong, what it should be, how many affected, business impact
- **Priority:** Urgent / High / Medium / Low
- **Assignee:** Engineering contact if known; otherwise leave for triage
- **Links:** Correction sheet, Slack thread, CITY token

---

## When to Use Linear vs. Slack

| Situation | Use |
|-----------|-----|
| Urgent, immediate issue | Slack `#tech-support` first, then Linear for tracking |
| Bug that needs engineering fix | Linear ticket (Sales & Transactions) |
| Cross-functional project | Linear project (AGENT or relevant team) |
| Quick config question | Slack DM to engineer |
