# CP2 — Cash Plus Enhancement Vision

**Status:** Planning
**What it is:** An enhanced version of Cash Plus with clearer agent expectations, improved training, and potential automation support.

---

## Problem Statement

The current Cash Plus program has several operational pain points:
- Contract generation bugs require manual correction
- Agent expectations aren't clearly defined at onboarding
- No systematic training for agents on CP-specific processes
- Corrections are manual and time-consuming
- No clear escalation channel or DRI for operational issues

CP2 is the opportunity to address these at the program design level, not just operationally.

---

## CP2 Enhancement Areas

### 1. Agent Training and Expectations
- Define clear expectations for CP agents (vs. standard Cash agents)
- Build training materials covering: what CP is, how the listing works alongside the cash offer, what happens if the listing sells first
- Delivery method TBD (likely: async Loom/doc + live Q&A option)

### 2. CITY Workflow Improvements
- Work with CITY engineering to reduce contract type errors at the source
- Potential: smarter defaults, validation before contract generation, clearer labels in the UI
- Feature requests to be submitted via `#tech-support`

### 3. Gumloop Automation Potential
- **Weekly update bot:** Auto-notify HSAs of CITY tooling updates for CP
- **New agent welcome:** Auto-send welcome email when a new CP agent is onboarded
- **Correction trigger:** Alert when a CITY correction is needed based on data mismatch
- All automation ideas go to the AI Automation Ideas backlog first

### 4. Clearer Escalation and DRI Structure
- Named DRI for CP operational bugs
- Defined escalation channel with SLA
- Clear runbook for corrections (this repo!)

### 5. Eric's Sheet Cleanup
- Tidy up the CP tracking sheet to reduce manual work and confusion
- Align column names with CITY terminology
- Remove stale data rows
