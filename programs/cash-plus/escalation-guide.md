# Cash Plus — Escalation Guide

---

## When to Escalate

| Situation | Action |
|-----------|--------|
| Single property CITY correction needed | Use correction sheet process — no escalation needed |
| Multiple properties with same bug | Flag as systematic — escalate to engineering + Mark |
| Unknown DRI for a CP bug | Raise in team channel, ask Mark to assign |
| Agent reports wrong product type on their listing | Verify in CITY first, then correct; communicate to agent |
| CP2 feature or behavior change needed | Document request → Slack → Linear |

---

## Escalation Paths

### Operational / Process Issues
**Who:** Program Manager (Tara)
**When:** Day-to-day corrections, individual property issues
**How:** Correction sheet + engineering Slack coordination

### Bug / Engineering Issues
**Who:** CITY engineering team
**Channel:** `#tech-support` Slack
**When:** Systematic contract generation errors, CITY behavior bugs
**What to include:** Property address, TXN ID, what's wrong, what it should be, how many properties affected

### Strategic / Program Issues
**Who:** Mark Biggins, Director Agent Partnerships
**When:** Retro needed, unclear ownership, partner-facing risk, product roadmap questions
**How:** 1:1, Partner Programs Weekly, or direct Slack

### Agent-Facing Issues
**Who:** Program Manager → agent communication
**When:** Agent has wrong contract type, commission affected, agent relationship at risk
**How:** Direct agent communication + CITY correction to align
