# Program Changes — Change Management

This section tracks changes to programs, documents key decisions, and provides templates for requesting and logging changes.

---

## Why We Document Changes

1. **Institutional memory** — when someone new joins or a question comes up 6 months later, we have context
2. **Accountability** — changes have owners and dates
3. **Audit trail** — for compliance, partner agreements, or retros
4. **Pattern recognition** — seeing what changed and why helps us make better decisions next time

## How to Log a Change

When a program change happens (process update, fee change, system change, partner term update):
1. Add a row to [change-log.md](change-log.md)
2. If it was a significant decision → also add to [decision-log.md](decision-log.md)
3. Update the relevant program docs in [../programs/](../programs/)

## How to Request a Change

Use [change-request-template.md](change-request-template.md) to document and communicate a change request before implementing it.
