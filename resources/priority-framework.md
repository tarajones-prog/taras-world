# Priority Framework — What to Work on Next

When everything feels urgent, use this framework to decide what deserves your attention first.

---

## The Four-Level Priority Stack

### Level 1 — Blocking Others
**Question:** Is anyone unable to move forward without me right now?

If yes → drop everything and handle it. Unblocking teammates and partners has a multiplier effect — your delay becomes their delay becomes a partner's delay.

Examples:
- ESO partner waiting on an executed document
- Engineer waiting on a correction sheet before they can fix a bug
- Agent waiting on payment confirmation before they'll refer the next deal

---

### Level 2 — Hard Deadlines
**Question:** Is there a commitment or deadline today or tomorrow?

If yes → do it next. Hard deadlines are non-negotiable. Missing them damages trust with partners and managers.

Examples:
- Assignment Agreement must go out ≤7 days before COE
- CE class due March 31 (NM license)
- Linear project deliverable due date

---

### Level 3 — Active Fires
**Question:** Is something broken, incorrect, or at risk of cascading?

If yes → address before it spreads. Active fires get worse if ignored — a CITY labeling bug affecting one property becomes many if left unfixed.

Examples:
- Cash Plus labeling bug affecting multiple properties
- ESO email group not connecting (blocking communications)
- Payment sent to wrong address

---

### Level 4 — Compounds Over Time
**Question:** What work, if done now, will make everything easier later?

Do this when Levels 1-3 are clear. These are the high-leverage activities that look optional but build the foundation for scale.

Examples:
- ESO pilot ops and process documentation → every new ESO deal is smoother
- Agent onboarding playbooks → every new ALO agent gets a better start
- Payment process improvements → fewer reconciliation headaches

---

## The "Nerd Out" Test

When considering a new idea or initiative:
- Is it in service of Levels 1-3 right now?
- Or is it a Level 4 investment that's worth prioritizing?
- Or is it a distraction?

Not everything that feels interesting is high-leverage. Apply the stack.

---

## Quick Decision Cheat Sheet

| Situation | Priority Level | Action |
|-----------|---------------|--------|
| Partner is waiting on me | 1 | Handle now |
| ESO assignment ≤7 days out | 1 + 2 | Handle immediately |
| Hard deadline today or tomorrow | 2 | Do this next |
| Bug affecting multiple properties | 3 | Fix before it spreads |
| No fires, no deadlines | 4 | ESO ops, process docs, automation |
| New interesting idea | Evaluate | Apply the four-level stack first |

---

## Protected Time

**NO MEETING WEDNESDAY** — this is protected focus time. Guard it fiercely. This is the day for Level 4 work — the compounding, high-leverage tasks that don't survive a meeting-heavy schedule.
